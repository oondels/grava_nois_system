## Objetivo da Sessao

Adicionar utilitario para converter `.env` legado em `config.json`, aproveitando a implementacao atual de configuracao persistente do sistema edge.

---

## O que foi Feito

### 11:59 Conversor de env legado para config gerenciada

**Acao**: Criado script executavel para ler um `.env` existente e gerar `config.json` com apenas parametros operacionais nao sensiveis.

**Arquivos alterados**:
- `env_to_config.sh` - converte `.env` para `config.json`, preservando segredos/identidade no env, convertendo RTSP com credenciais para `env:VAR_NAME` e oferecendo `--dry-run`.
- `README.md` - documentado uso local e uso em host legado provisionado por `grava_nois_config`.
- `docs/specs/system/CONFIGURATION.md` - documentada migracao via script.
- `CHANGELOG.md` - registrado utilitario de migracao.
- `tests/test_legacy_compatibility.py` - reset do cache do loader entre cenarios com env alterado.
- `tests/test_mqtt_settings.py` - reset do cache do loader entre cenarios com env alterado.
- `tests/test_capture_ffmpeg_command.py` - reset do cache do loader para tuning RTSP por env.
- `tests/test_mobile_format.py` - reset do cache do loader para MOBILE_FORMAT/VERTICAL_FORMAT por env.
- `tests/test_no_file_collisions.py` - reset do cache do loader para cenarios multi-camera por env.
- `tests/test_multi_camera_settings.py` - reset do cache do loader para cenarios multi-camera por env.
- `tests/test_worker_multi_camera.py` - reset do cache do loader para cenarios de worker com env alterado.

**Resultado**: Devices legados podem gerar um `config.json` inicial a partir do `.env` atual sem expor tokens, senhas, `DEVICE_SECRET` ou URLs RTSP com credenciais em texto plano.

## Raciocinio e Bases

### Hipotese seguida

Como `config.json` ja estava implementado, o caminho de menor risco foi criar um conversor externo e idempotente, mantendo env como fonte de segredos e fallback.

### Referencias
- `docs/specs/system/CONFIGURATION.md`: contrato do schema e politica de precedencia.
- `src/config/config_loader.py`: campos suportados e regras de segredos.
- `src/config/config_schema.py`: ranges e validacao do JSON gerado.

### Trade-offs avaliados
- Migrar segredos vs manter em env: mantidos em env para evitar vazamento e preservar compatibilidade HMAC/MQTT.
- Escrever script shell puro vs shell com Python embutido: usado Python embutido para parse robusto de `.env`, JSON e `GN_CAMERAS_JSON`.
