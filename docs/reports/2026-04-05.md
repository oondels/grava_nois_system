**Task**: Implementar a fase 1 de presenca MQTT no `grava_nois_system`, com heartbeat, estado resumido e estrutura futura de command/control sem ativar comandos remotos.

### PRE-DEV REPORT: feat(edge): adicionar presenca MQTT isolada do pipeline principal

**GOAL**
Adicionar uma camada dedicada de MQTT no edge para publicar presenca e heartbeat do device sem misturar essa responsabilidade com captura, buffer, worker e upload.

**DEFINITION OF DONE**
1. O edge deve ter uma estrutura dedicada em `src/services/mqtt/` com cliente MQTT, presenca e placeholders de comandos.
2. MQTT deve ser inicializado de forma opcional, isolada e resiliente, sem derrubar captura, trigger ou worker quando o broker estiver indisponivel.
3. A presenca deve publicar ao menos `device_id`, `client_id`, `venue_id`, `status`, `agent_version`, `timestamp`, `last_seen` e um estado resumido do runtime.
4. Deve existir namespace de topicos consistente para `presence`, `heartbeat`, `state` e futuros `commands`.
5. Nenhum comando remoto pode ser executado nesta fase.
6. `.env.example`, `README.md`, `CHANGELOG.md` e specs do edge devem refletir a nova capacidade.
7. Devem existir testes automatizados para config/payload/lifecycle base do modulo MQTT.
